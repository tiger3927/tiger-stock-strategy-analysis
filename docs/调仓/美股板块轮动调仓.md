## 1. 核心理念

### 1.1 什么是板块轮动

板块轮动是一种**主动组合管理策略**，根据经济周期阶段在不同行业板块之间动态配置资金，超配有望跑赢的板块，低配可能跑输的板块。

### 1.2 与追涨杀跌的本质区别

| 维度 | 追涨杀跌 | 机构级轮动 |
|------|---------|-----------|
| 决策依据 | 当日涨幅榜 | 多因子综合评分 + 相对强弱 |
| 时间窗口 | 日线级别 | 周/月线级别中期动量 |
| 仓位管理 | 满仓进出 | 分批加减、资金闭环 |
| 风控 | 无 | 硬性仓位上限、止损规则 |
| 执行节奏 | 一次性 | 分多日拆单执行 |

### 1.3 核心原则（不可违反）

1. **永远先卖弱，再买强** — 不同时进行买卖操作，资金必须闭环
2. **分批执行** — 一轮调仓至少分 2-3 天完成，不单日大规模换仓
3. **单板块上限 20%，单票上限 5%** — 硬性风控红线
4. **中期动量优于短期爆发** — 6-12 个月相对强弱比 5 日涨幅更有意义
5. **宏观是总开关** — 大盘系统性风险时只卖不买

## 3. 第一层：宏观环境定基调

### 3.1 经济周期四阶段与板块映射

| 周期阶段 | 宏观特征 | 领涨板块 | 防御板块 | 核心指标 |
|---------|---------|---------|---------|---------|
| **早期复苏** | 利率下降、信用扩张、PMI 回升 | 金融(XLF)、消费可选(XLY)、工业(XLI)、材料(XLB) | — | 收益率曲线斜率、PMI 拐点 |
| **中期扩张** | 稳定增长、温和通胀 | 科技(XLK)、能源(XLE)、工业(XLI) | — | GDP 增速、失业率 |
| **晚期过热** | 通胀上升、加息、增长放缓 | 能源(XLE)、材料(XLB) | 必需消费(XLP)、医疗(XLV)、公用事业(XLU) | CPI、美联储利率决议 |
| **衰退** | GDP 负增长、失业上升 | — | 公用事业(XLU)、必需消费(XLP)、医疗(XLV) | 信用利差、失业率 |

### 3.2 宏观信号监测清单

| 信号类别 | 具体指标 | 数据来源 | 更新频率 |
|---------|---------|---------|---------|
| 利率 | 美联储基准利率、2Y/10Y 利差 | FRED | 每日 |
| 信用 | 投资级/高收益信用利差 | FRED / Bloomberg | 每日 |
| 经济景气 | ISM 制造业/非制造业 PMI | ISM | 每月 |
| 就业 | 非农就业、失业率、初请失业金 | BLS | 每周/每月 |
| 通胀 | CPI、PPI、PCE | BLS / BEA | 每月 |
| 市场情绪 | Fear & Greed Index | CNN Money | 每日 |

### 3.3 宏观状态判定规则

```
IF 收益率曲线陡化 AND PMI 连续 2 月回升 AND 信用利差收窄:
    周期阶段 = 早期复苏
ELIF GDP 正增长 AND 失业率低位 AND 通胀温和:
    周期阶段 = 中期扩张
ELIF CPI > 3% AND 美联储加息 AND 收益率曲线平坦化:
    周期阶段 = 晚期过热
ELIF GDP 负增长 AND 失业率上升 AND 信用利差扩大:
    周期阶段 = 衰退
```

### 3.4 大盘总开关规则

```
IF 大盘处于系统性风险（S&P 500 跌破 200 日均线 OR VIX > 30）:
    操作模式 = 防御模式（只卖不买，逐步减仓至 50% 以下）
ELIF 大盘震荡（S&P 500 在 50-200 日均线之间）:
    操作模式 = 正常轮动（小范围调仓）
ELIF 大盘趋势向上（S&P 500 在 50 日均线上方）:
    操作模式 = 积极模式（适度加仓强势板块）
```

## 4. 第二层：板块评分与排序

### 4.1 板块分类体系

采用 GICS 11 大行业板块，对应 ETF 如下：

| 板块 | ETF 代码 | 周期属性 |
|------|---------|---------|
| 信息技术 | XLK | 成长/周期 |
| 金融 | XLF | 周期 |
| 医疗保健 | XLV | 防御 |
| 消费可选 | XLY | 周期 |
| 消费必需 | XLP | 防御 |
| 能源 | XLE | 周期 |
| 工业 | XLI | 周期 |
| 材料 | XLB | 周期 |
| 公用事业 | XLU | 防御 |
| 房地产 | XLRE | 防御 |
| 通信服务 | XLC | 成长 |

### 4.2 多因子评分模型

```
板块综合得分 = 0.30 × 动量因子 + 0.25 × 经济周期因子
              + 0.20 × 价值因子 + 0.15 × 质量因子 + 0.10 × 情绪因子
```

#### 4.2.1 动量因子（权重 30%）

| 子指标 | 权重 | 计算方式 |
|-------|------|---------|
| 6 个月相对强弱 (RS) | 40% | 板块价格 / SPY 价格，取 6 个月变化率 |
| 3 个月相对强弱 (RS) | 30% | 同上，3 个月窗口 |
| 12 个月相对强弱 (RS) | 20% | 同上，12 个月窗口 |
| 20 日 ROC | 10% | 20 日变化率，捕捉短期趋势 |

**RS 计算**：
```
RS = 板块 ETF 价格 / SPY 价格
RS_ROC = (RS_today / RS_n_days_ago - 1) × 100
```

**信号质量确认（三位一体）**：
- 价格突破：板块价格突破关键阻力位
- RS 突破：RS 曲线同步走强
- 市场广度：板块内大部分成分股上涨（>60%）

#### 4.2.2 经济周期因子（权重 25%）

根据当前宏观阶段，给板块打分：

| 板块 | 早期复苏 | 中期扩张 | 晚期过热 | 衰退 |
|------|---------|---------|---------|-----|
| 金融 (XLF) | +2 | +1 | -1 | -2 |
| 科技 (XLK) | +1 | +2 | 0 | -2 |
| 消费可选 (XLY) | +2 | +1 | -1 | -2 |
| 工业 (XLI) | +2 | +1 | 0 | -1 |
| 材料 (XLB) | +1 | +1 | +1 | -2 |
| 能源 (XLE) | -1 | +1 | +2 | -1 |
| 医疗 (XLV) | 0 | 0 | +1 | +2 |
| 消费必需 (XLP) | -1 | 0 | +1 | +2 |
| 公用事业 (XLU) | -2 | -1 | +1 | +2 |
| 房地产 (XLRE) | -1 | 0 | 0 | +1 |
| 通信 (XLC) | +1 | +1 | 0 | -1 |

#### 4.2.3 价值因子（权重 20%）

| 子指标 | 权重 | 计算方式 |
|-------|------|---------|
| 板块 P/E 历史分位 | 50% | 当前 PE 在 5 年历史中的百分位（越低越好） |
| 板块 P/B 历史分位 | 30% | 当前 PB 在 5 年历史中的百分位 |
| 股息率 | 20% | 板块股息率相对历史水平 |

#### 4.2.4 质量因子（权重 15%）

| 子指标 | 权重 | 计算方式 |
|-------|------|---------|
| ROE 中位数 | 40% | 板块内个股 ROE 中位数 |
| 盈利增长稳定性 | 30% | 过去 3 年盈利增长的标准差（越低越好） |
| 负债率 | 30% | 板块整体 D/E 比率（越低越好） |

#### 4.2.5 情绪因子（权重 10%）

| 子指标 | 权重 | 计算方式 |
|-------|------|---------|
| 资金流 | 50% | 板块 ETF 过去 5 日资金净流入 |
| 期权偏度 | 30% | 板块看跌/看涨期权比率 |
| 分析师情绪 | 20% | 板块内上调/下调评级比率 |

### 4.3 板块排序与分层规则

```
每周/每月对所有 11 个板块进行综合评分并排序：

强势板块（Top 3-4）：综合得分最高 → 超配 / 加仓
中性板块（中间 4-5）：综合得分中等 → 维持权重
弱势板块（Bottom 3）：综合得分最低 → 低配 / 减仓

附加条件：
- 强势板块必须满足 RS 曲线处于上行趋势
- 弱势板块若连续 2 周排名 Bottom 3 → 强制减仓
- 弱势板块若连续 3 周排名 Bottom 3 → 强制清仓
```

## 5. 第三层：子板块评分体系（GICS Level 2）

### 5.1 为什么需要子板块粒度

GICS Level 1（11 个 Sector）在某些场景下粒度太粗：

```
示例：2023-2024 AI 行情
  信息技术 (XLK) 整体强势
  但内部：半导体设备 +80%，软件 +20%，硬件 +10%
  如果只按 XLK 评分 → 整个板块都加仓
  如果按子板块评分 → 只加仓半导体，不加仓软件/硬件
```

**子板块轮动的核心价值**：在强势板块中精确定位最强细分领域，在弱势板块中识别可能逆势的细分赛道。

### 5.3 双层评分模型

```
最终子板块得分 = 0.50 × 所属 Sector 得分 + 0.50 × 子板块独立得分
```

#### 5.3.1 子板块独立得分

与 Sector 评分使用相同的多因子模型，但数据源为子板块级别：

```
子板块独立得分 = 0.30 × 子板块动量 + 0.25 × 经济周期因子
               + 0.20 × 子板块价值 + 0.15 × 子板块质量 + 0.10 × 子板块情绪
```

**动量因子计算**：

```
子板块 RS = 子板块价格（等权合成）/ SPY 价格
子板块 RS_ROC = (RS_today / RS_n_days_ago - 1) × 100
```

对于没有独立 ETF 的子板块，使用成分股等权合成价格指数：

```
子板块价格 = 成分股价格等权平均
成分股选择：该子板块内市值 Top 5 或流动性 Top 5 的个股
```

#### 5.3.2 双层评分决策矩阵

| Sector 评分 | 子板块评分 | 最终决策 |
|------------|-----------|---------|
| 强势 (Top 3-4) | 强势 | **强烈加仓** — 板块和子板块共振 |
| 强势 (Top 3-4) | 弱势 | **仅保留，不加仓** — 板块强但细分弱 |
| 弱势 (Bottom 3) | 强势 | **保留底仓观察** — 子板块可能逆势 |
| 弱势 (Bottom 3) | 弱势 | **优先减仓/清仓** — 双重弱势 |

### 5.4 子板块排序与分层

```
对所有 25 个子板块进行评分并排序：

强势子板块（Top 6-8）：综合得分最高 → 超配
中性子板块（中间 10-12）：维持权重
弱势子板块（Bottom 5-7）：低配 / 减仓

附加规则：
- 子板块评分不得高于其所属 Sector 评分超过 1 个标准差
  （防止子板块过度偏离板块基本面）
- 若 Sector 为弱势但子板块为强势 → 子板块仓位上限减半
```

### 5.5 子板块轮动的典型场景

```
场景一：科技板块 (XLK) 内部轮动
  Sector 评分：科技 = 强势（排名第 2）
  子板块评分：
    - 半导体与半导体设备 = 强势（排名第 1）
    - 软件与服务 = 中性（排名第 10）
    - 技术硬件与设备 = 弱势（排名第 20）
  操作：加仓半导体，维持软件，减仓硬件

场景二：医疗板块 (XLV) 内部轮动
  Sector 评分：医疗 = 中性（排名第 6）
  子板块评分：
    - 制药与生物科技 = 强势（排名第 5）
    - 医疗设备与服务 = 弱势（排名第 18）
  操作：加仓生物科技，减仓医疗设备
```

## 6. 第四层：持仓映射与分层

### 6.1 个股 → 子板块映射

将每个持仓个股映射到 GICS Level 2 子板块：

```
输入：持仓列表 [AAPL, JPM, XOM, JNJ, ...]
输出：双层映射表
  Sector 层：
    - 科技: [AAPL, MSFT, NVDA, ...]
    - 金融: [JPM, GS, BAC, ...]
    - ...
  Industry Group 层：
    - 半导体与设备: [NVDA, AMD, INTC, ...]
    - 软件与服务: [MSFT, ORCL, CRM, ...]
    - 银行: [JPM, BAC, WFC, ...]
    - 制药与生物科技: [JNJ, PFE, MRK, ...]
    - ...
```

### 6.2 双层持仓分层

根据 Sector 评分 + 子板块评分，将所有持仓分为五层：

| 层级 | Sector | 子板块 | 操作策略 |
|------|--------|--------|---------|
| **S1 强烈加仓** | 强势 | 强势 | 优先加仓，最高权重 |
| **S2 适度加仓** | 强势 | 中性 | 小幅加仓优质标的 |
| **S3 维持** | 中性 | 任意 | 维持权重，不做调整 |
| **S4 观察减仓** | 弱势 | 强势 | 保留底仓，不加仓 |
| **S5 优先减仓** | 弱势 | 弱势 | 阶梯减仓/清仓 |

### 6.3 个股层面的二次筛选

在子板块内部，对个股进行二次评分：

```
个股评分 = 0.30 × Sector 得分 + 0.30 × 子板块得分
         + 0.20 × 个股动量 + 0.15 × 个股质量 + 0.05 × 个股流动性

S1 层级内：
  - 个股评分 Top 60% → 可加仓
  - 个股评分 Bottom 40% → 仅保留，不加仓

S5 层级内：
  - 个股评分 Top 20% → 保留底仓观察
  - 个股评分 Bottom 80% → 优先减仓/清仓
```

### 6.4 个股选股池与分层

#### 6.4.1 个股 → 子板块映射

个股归属 GICS 子板块的确定方式（按优先级）：

```
方式一：GICS 官方分类（最准确）
  - 数据源：Bloomberg、FactSet、MSCI 的 GICS 分类接口
  - 每个股票有唯一的 GICS 代码（8 位数字，如 45301020 = 半导体）
  - 每季度更新一次（GICS 年度调整在 3 月/9 月）

方式二：ETF 成分股反推（次优）
  - 子板块 ETF（如 SMH）的成分股 = 该子板块的核心股票
  - 适用于有独立 ETF 的子板块

方式三：行业关键词匹配（兜底）
  - 通过公司财报中的 SIC/NAICS 代码或业务描述匹配
  - 精度较低，仅作为补充
```

#### 6.4.2 三层选股池

对每个 GICS 子板块，维护一个 **分层选股池**：

| 层级 | 定位 | 数量 | 操作策略 |
|------|------|------|---------|
| **Tier 1 龙头股** | 板块风向标，流动性最好 | 3-5 只 | 板块配置的主要工具，加仓/减仓首选 |
| **Tier 2 中盘成长** | 超额收益来源，弹性好 | 5-10 只 | 板块强势时优先加仓 |
| **Tier 3 潜力小盘** | 高赔率品种，高风险 | 5-15 只 | 总仓位 ≤ 5%，单只 ≤ 1% |

#### 6.4.3 选股评分公式

```
个股综合评分 = 0.25 × 子板块得分 + 0.20 × 个股动量
             + 0.15 × 个股质量 + 0.15 × 个股估值
             + 0.10 × 个股流动性 + 0.10 × 催化剂信号
             + 0.05 × 小盘溢价（仅 Tier 3 适用）
```

## 7. 第五层：调仓执行流程

### 7.1 总体流程

```
Step 1: 宏观判定 → 确定操作模式（积极/正常/防御）
Step 2: 板块评分 → 输出 11 板块排序
Step 3: 持仓映射 → 个股映射到板块，分层
Step 4: 先处理弱势持仓（减仓/清仓）
Step 5: 计算释放资金
Step 6: 再处理强势持仓（加仓/新开）
Step 7: 风控校验
Step 8: 拆单执行
```

### 7.2 处理弱势持仓（先卖）

**原则：先卖后买，资金闭环，绝不同时操作**

#### 7.2.1 减仓节奏

```
Day 1: 弱势板块持仓统一减仓 30%-50%
Day 2: 若板块评分仍为弱势 → 再减 30%（剩余仓位的 30%）
Day 3: 若板块评分连续 3 周 Bottom 3 → 清仓
```

#### 7.2.2 减仓优先级

```
优先减仓顺序：
1. 板块评分最低的板块中的个股
2. 个股评分最低的个股
3. 仓位最重的个股（降低集中度）
4. 流动性最好的个股（容易卖出）
```

#### 7.2.3 特殊情况处理

```
IF 个股处于盈利状态:
    正常减仓，锁定利润
IF 个股处于亏损状态:
    若板块持续弱势 → 止损减仓（不因亏损而持有）
    若板块只是短期回调 → 可保留底仓观察
IF 个股有重大利好（财报超预期、新产品获批等）:
    即使板块弱势，可保留底仓
```

### 7.3 处理强势持仓（后买）

#### 7.3.1 资金分配

```
释放资金分配规则：
1. 只流入当天排名 Top 2-3 的强势板块
2. 每个强势板块只加仓 1-2 只个股
3. 优先加仓已持有的优质标的（不开过多新仓）
4. 新开仓不超过 2 只（防止持仓数量爆炸）
```

#### 7.3.2 加仓节奏

```
Day 1: 使用释放资金的 50% 进行加仓
Day 2: 若板块仍强势 → 使用 30%
Day 3: 若板块仍强势 → 使用 20%
（剩余资金保留为现金缓冲）
```

#### 7.3.3 买入限制

```
买入限制条件：
- 不追高：个股当日涨幅 > 3% 时不买入
- 不买暴涨过的：个股 5 日涨幅 > 15% 时不买入
- 回踩买入：优先在个股回踩 20 日均线时买入
- 分 3 档建仓：30% / 30% / 40%，分批入场
```

### 7.4 单日调仓示例

```
假设你有 40 只持仓，分布在 11 个板块：

板块评分结果：
  强势板块：科技(XLK)、金融(XLF)、工业(XLI)
  弱势板块：能源(XLE)、材料(XLB)、房地产(XLRE)

持仓映射：
  弱势板块持仓：12 只（能源 4 只 + 材料 5 只 + 房地产 3 只）
  中性/强势板块持仓：28 只

AI 当日执行：
  1. 12 只弱势持仓 → 统一减仓 40%
  2. 释放资金约 15% 总仓位
  3. 从释放资金中，分配：
     - 科技板块：加仓已持有的 2 只优质股
     - 金融板块：加仓已持有的 1 只 + 新开 1 只
  4. 新增持仓不超过 2 只
  5. 总仓位保持稳定（约 85%-90%）
  6. 剩余资金保留为现金缓冲（10%-15%）
```

## 8. 第六层：仓位约束与风控

### 8.1 硬性仓位上限

| 约束项 | 上限 | 说明 |
|-------|------|------|
| 单个板块 (Sector) 总仓位 | ≤ 20% | 防止板块系统性风险 |
| 单个子板块 (Industry Group) 总仓位 | ≤ 12% | 防止子板块过度集中 |
| 单只个股仓位 | ≤ 5% | 防止个股黑天鹅 |
| Tier 3 小盘股总仓位 | ≤ 5% | 小盘股高风险，总量控制 |
| 单只 Tier 3 小盘股仓位 | ≤ 1% | 单只小盘股严格限制 |
| 前 3 大板块合计 | ≤ 50% | 防止过度集中 |
| 现金最低比例 | ≥ 5% | 保持流动性 |
| 最大持仓数量 | ≤ 50 只 | 管理复杂度上限 |

### 8.2 止损规则

| 触发条件 | 操作 |
|---------|------|
| 个股亏损 > 8% | 强制止损 |
| 板块评分连续 2 周 Bottom 3 | 板块内持仓减仓 50% |
| 板块评分连续 3 周 Bottom 3 | 板块内持仓清仓 |
| S&P 500 跌破 200 日均线 | 总仓位降至 50% 以下 |
| VIX > 30 | 停止所有买入操作 |

### 8.3 流动性风控

```
单日单只个股买卖金额 ≤ 该股日均成交额 × 1%
（防止冲击成本过大）

示例：
  AAPL 日均成交额 = 100 亿美元
  单日最大买卖 AAPL 金额 = 1 亿美元（对散户 AI 可放宽）
```

### 8.4 再平衡触发条件

```
满足以下任一条件即触发再平衡：
1. 月度固定再平衡（每月第一个交易日）
2. 板块评分发生重大变化（Top 3 板块变更）
3. 子板块评分发生重大变化（Top 6 子板块变更超过 3 个）
4. 宏观阶段切换信号出现
5. 单板块仓位超过 20% 上限
6. 单子板块仓位超过 12% 上限
7. 大盘进入防御模式
```

## 9. 第七层：拆单与执行优化

### 9.1 拆单策略

| 订单规模 | 拆单方式 | 执行周期 |
|---------|---------|---------|
| 小单（< 5% 日均成交额） | 一次性市价单 | 当日 |
| 中单（5%-20% 日均成交额） | 分 3-5 笔，每 30 分钟一笔 | 当日 |
| 大单（> 20% 日均成交额） | 分 2-3 天，每天均匀拆单 | 2-3 天 |

### 9.2 执行时间窗口

```
美股最佳执行时段（美东时间）：
- 开盘后 30 分钟（9:30-10:00）：流动性最好，但波动大
- 午盘（11:00-14:00）：流动性稳定，适合拆单
- 收盘前 30 分钟（15:30-16:00）：机构调仓密集，流动性好

避免执行时段：
- 开盘前 15 分钟（9:30-9:45）：波动剧烈，价差大
- 午休前后（12:00-12:30）：流动性下降
```

### 9.3 滑点控制

```
预期滑点 = 订单规模 / 日均成交额 × 平均价差 × 2

IF 预期滑点 > 0.3%:
    缩小单笔订单规模
    延长执行周期
    改用限价单
```

## 10. 双周期再平衡机制

### 10.1 月度大再平衡

```
每月第一个交易日执行：
1. 全面重新计算 11 板块多因子评分
2. 全面重新计算 25 子板块多因子评分
3. 重新排序并确定强势/中性/弱势板块及子板块
4. 全面调整板块及子板块权重至目标配置
5. 调仓幅度控制在 ±10% 以内（避免过度换手）
```

### 10.2 周度小幅微调

```
每周五执行：
1. 更新板块评分（仅更新动量因子和情绪因子）
2. 更新子板块评分（仅更新动量因子和情绪因子）
3. 检查是否有板块排名发生重大变化
4. 检查是否有子板块排名发生重大变化
5. 如有板块从强势跌出 Top 5 → 小幅减仓
6. 如有板块新进入 Top 3 → 小幅加仓
7. 如有子板块从强势跌出 Top 8 → 小幅减仓
8. 单次调仓幅度不超过 ±3%
```

### 10.3 异常事件触发

```
以下事件触发即时评估（非固定周期）：
1. 美联储利率决议
2. 重大经济数据发布（非农、CPI、GDP）
3. 地缘政治事件
4. 板块内重大并购/监管变化
```

## 12. 常见失败陷阱

### 12.1 陷阱一：追涨杀跌

**表现**：看到板块大涨就追入，大跌就割肉。

**对策**：
- 只用相对强弱（RS）而非绝对涨幅做决策
- 设置确认条件：RS 曲线必须处于上行趋势
- 买入不追高（当日涨幅 > 3% 不买）

### 12.2 陷阱二：过度交易

**表现**：每天大规模调仓，换手率过高。

**对策**：
- 固定月度大再平衡 + 周度微调的双周期机制
- 单次调仓幅度限制（月度 ±10%，周度 ±3%）
- 计算交易成本，纳入决策

### 12.3 陷阱三：滞后指标陷阱

**表现**：等经济数据确认后才调仓，已经晚了。

**对策**：
- 使用领先指标（收益率曲线、信用利差）而非滞后指标（GDP）
- 关注价格信号（价格是最快的领先指标）
- 分批调仓，边确认边调整

### 12.4 陷阱四：过度集中

**表现**：重仓押注一两个板块。

**对策**：
- 硬性单板块上限 20%
- 前 3 大板块合计不超过 50%
- 定期检查集中度

### 12.5 陷阱五：忽视交易成本

**表现**：频繁换手导致交易成本侵蚀收益。

**对策**：
- 月度再平衡为主，减少非必要交易
- 大单拆分为小单执行
- 计算预期滑点，超过阈值则调整

### 12.6 陷阱六：对抗长期趋势

**表现**：在长期下行趋势的板块中反复抄底。

**对策**：
- 尊重长期趋势（200 日均线方向）
- 弱势板块连续 3 周 Bottom 3 必须清仓
- 不逆势加仓
