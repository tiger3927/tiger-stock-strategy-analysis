> 适用阶段：网格管理期
> 路由文档：[00-Router.md](00-Router.md)

## 一、网格机制理解

### 1.1 两层资金结构

| 层级 | 用途 | 占比 | 操作规则 |
|:--|:--|:--|:--|
| **基础仓位** | 建立策略的底仓锚，是网格运行的基准线和利润来源 | 建议首仓占比 | 仅在大幅反转或收割期时卖出 |
| **网格资金** | 围绕基础仓进行分档加仓和减仓，持续摊低（做多）/ 摊高（做空）持仓成本 | 最大仓位 - 首仓 | 网格自动加减，AI 不干预单次操作 |

> **趋势锁仓模式：** 当 AI 判断真实趋势正在展开时，可关闭网格减仓（`是否允许网格减仓: false`）。此时网格资金的角色从"震荡收割器"临时转为"趋势仓位"，与基础仓一起锁住吃波段。趋势减弱后恢复减仓，网格资金回到震荡收割角色。详见 §三-7 和 §十的趋势锁仓模板。

### 1.2 网格运行逻辑

**做多方向：**

```
价格向下突破第一档 → 第一次加仓（网格资金池消耗 1/N）
价格继续向下突破第二档 → 第二次加仓
...
价格向上回到盈利线 → 最后一次加仓的部分减仓，盈利落入网格
价格继续向上 → 逐档减仓，锁定更多利润
```

**做空方向（方向对称，逻辑相反）：**

```
价格向上突破第一档 → 第一次加空仓（网格资金池消耗 1/N）
价格继续向上突破第二档 → 第二次加空仓
...
价格向下回到盈利线 → 最后一次加空仓的部分减仓（买回平仓），盈利落入网格
价格继续向下 → 逐档减空仓，锁定更多利润
```

网格的本质：**用多档资金接住不利波动，用少量有利波动即可实现整体盈利。**

### 1.3 基础底仓的不可侵犯性

基础底仓不是网格的一部分，是网格的锚。做多时卖出 / 做空时买入平仓，仅限以下六种情况：

1. 网格充分运行，进入利润收割期
2. 基本面发生重大恶化（做多）/ 重大好转（做空）
3. 趋势高置信度反转（置信度 > 0.75）
4. 亏损达到止损条件
5. 出现重大不可控事件风险
6. 策略整体进入风险退出期

**凡不满足以上任一条件——不得卖出基础底仓（做多）或买入平仓基础空仓（做空）。**

## 二、网格间距计算规则

网格间距决定价格朝不利方向（做多=下跌、做空=上涨）波动多少触发一次补仓。

### 2.1 计算公式

做多时用支撑位维度，做空时用压力位维度：

```
做多：
网格间距 = max(0.015, min(
    ATR(14) / 当前价格 × 1.2,
    (一级支撑位 - 二级支撑位) / 一级支撑位 / 3
))

做空：
网格间距 = max(0.015, min(
    ATR(14) / 当前价格 × 1.2,
    (二级压力位 - 一级压力位) / 一级压力位 / 3
))
```

### 2.2 变量说明

| 变量 | 说明 |
|:--|:--|
| ATR(14) | 14日平均真实波幅（美元） |
| ATR(14) / 当前价格 × 1.2 | ATR 法间距上限（波动率跟随） |
| (一级支撑位 - 二级支撑位) / 一级支撑位 / 3 | 做多支撑法间距上限（结构约束） |
| (二级压力位 - 一级压力位) / 一级压力位 / 3 | 做空压力法间距上限（结构约束） |
| 下限 1.5% | 防止极端低波动品种频繁误触发 |

### 2.3 设计逻辑

公式取 ATR 法和结构法的较小值：
- **ATR 维度**：高波动品种间距更宽，低波动品种间距更窄
- **结构法维度**：做多时保证一二级支撑位之间至少容纳 3 档网格；做空时保证一二级压力位之间至少容纳 3 档网格
- **下限 1.5%**：防止极端低波动品种间距过窄

### 2.4 示例（做多）

```
NVDA：ATR=$6.26，价格=$197，支撑 S1=$194，S2=$182
ATR法间距：6.26/197×1.2=3.81%
支撑法间距：(194-182)/194/3=2.06%
min(3.81%, 2.06%)=2.06% → 网格间距=2.06%
```

## 三、网格管理期中 AI 核心原则

策略阶段：`网格管理期`

**只看不碰。** 不干预单个网格的加减仓。仅从宏观层面判断：

1. 品种是否仍值得继续运行？
2. 需要调降最大仓位占比吗？（标的质量恶化但未到退出程度）
3. 需要调整网格间距吗？（ATR 或支撑结构显著变化，重新按公式计算。**间距调整时数量联动，保持总网格深度不变**：`新数量 = round(原间距 × 原数量 / 新间距)`，取整到最近的整数）
4. 是否进入利润收割期？（任一触发：价格优先 5 / 网格充分 5a / 利润达标 5b，详见 [00-Router.md](00-Router.md) 流转表。综合浮盈计算参见 [03-Exit-Rules.md §2.1](03-Exit-Rules.md)）
5. 是否进入风险退出期？（重大风险信号出现，含先行预警指标）
6. **当前价格是否已接近关键位？** 做多时距一级压力位 < 3% 且综合浮盈 ≥ 5%，或做空时距一级支撑位 < 3% 且综合浮盈 ≥ 5%——若是，在理由中明确提醒「价格已接近关键位，建议关注是否进入利润收割期」，并检查是否满足流转 5（价格优先）条件
7. **是否需要启动趋势锁仓？** 当 AI 综合技术面、外部信息（分析师共识、板块联动、宏观环境、实质性利好等）判断一波真实趋势正在展开——而非震荡区间内的随机假突破——可设置 `是否允许网格减仓: false`。效果是基础仓 + 所有网格仓锁在一起吃趋势，资金利用率从基础仓的 20-25% 跃升到整体 75-85%。判断要点：（1）趋势有延续性的实质支撑，不仅仅是几个技术指标好看；（2）价格在支撑-压力结构中还有充分运行空间；（3）对比"让网格照常收割"和"锁仓赌趋势"在当前市况下哪个更合理。当趋势出现减弱迹象（横盘停滞、外部信息转向、利润大幅回吐、逼近关键位等），应及时恢复网格减仓，让系统正常收割锁仓期间积累的浮盈

**网格子状态说明：** 网格管理期阶段内部包含两种子状态——「未触发」（网格尚未首次触发）和「已触发」（已至少触发一次）。两种子状态的 AI 操作约束完全一致，仅网格系统的分仓止盈是否自动执行有区别。AI 不应关注子状态切换，由网格系统自行管理。

## 四、离场诱因分级处理

以下诱因不应自动触发离场，但部分情况需升级为「警惕」并启动综合评估：

| 类别 | 诱因 | 级别 | 处理规则 |
|:--|:--|:--|:--|
| 盈利诱惑 | 小幅浮盈（< 5%） | 🟢 忽略 | 网格管理期阶段，浮盈来源可能是底仓自然波动或网格分仓止盈，均不到收割条件 |
| 盈利诱惑 | 中等浮盈（≥ 5%）且：做多时距一级压力位 < 3%，做空时距一级支撑位 < 3% | 🟡 警惕 | 需评估是否满足流转 5/5a/5b 条件，启动收割评估 |
| 亏损恐惧 | 普通回撤 | 🟢 忽略 | 这是马丁格尔等待加仓的正常波动，不是反转信号 |
| 信息噪声 | 非权威来源的短期新闻噪声 | 🟢 忽略 | 单条新闻不改变趋势结构 |
| 信息噪声 | 权威来源新闻 + 已引发价格异动 > 3% | 🟡 警惕 | 关注是否演变为基本面变化，需持续跟踪 |
| 指标波动 | 单一技术指标轻微转弱 | 🟢 忽略 | 指标在震荡区反复是常态 |
| 指标波动 | ≥ 3 个技术指标同步转弱 | 🟡 警惕 | 多指标共振可能是趋势变化的早期信号 |
| 预测变化 | 短线涨跌预测变化、置信度小幅下降 | 🟢 忽略 | 短期置信度波动是统计误差 |
| 操作冲动 | 单个网格盈利 | 🟢 忽略 | 网格盈利说明策略在工作，不是离场信号 |
| 数据异常 | 数据短暂缺失、价格数据异常 | 🟢 忽略 | 等数据恢复再判断 |

**🟡 警惕 ≠ 离场。** 警惕级仅表示需要启动综合评估，最终决策仍需对照退出条件判断（[03-Exit-Rules.md](03-Exit-Rules.md)）。若警惕信号同时出现 ≥ 2 个，应升级为 🔴 高风险，优先排查退出条件。

## 五、常见误判场景

### 做多方向

1. **「价格涨了 3%，赚了，卖了吧」** → 错误。3% 不是马丁格尔的收割目标。网格还没充分发挥。
2. **「价格跌了 2%，趋势可能变了，先止损」** → 错误。2% 回撤在马丁格尔中是被期待的加仓机会，不是止损信号。

### 做空方向

3. **「价格跌了 3%，做空赚了，平了吧」** → 错误。3% 不是马丁格尔的收割目标。网格还没充分发挥。
4. **「价格涨了 2%，做空趋势可能反了，先止损」** → 错误。2% 反弹在马丁格尔做空中是被期待的加空仓机会，不是止损信号。

### 通用

5. **「新闻说这个板块被监管调查，先跑吧」** → 需判断。传闻/讨论非正式政策 → 噪声。正式公告/法规改变 → 可考虑风险退出。
6. **「成交量萎缩，市场不关注这个品种了」** → 错误。缩量震荡对马丁格尔是中性偏好的环境。

## 六、止盈止损模式

### 6.1 五种模式定义

| 模式 | 含义 |
|:--|:--|
| 普通止盈 | 整体持仓达到指定盈利幅度后自动平仓 |
| 移动止盈 | 盈利达到启动线后，价格从最高点回撤指定幅度时自动平仓 |
| 网格分仓止盈 | 仅对网格加仓部分的单个网格进行止盈，基础底仓不受影响 |
| 止损 | 亏损达到指定幅度后自动平仓 |
| 整体平仓 | 无条件全部清仓（风险退出期专用） |

### 6.2 各阶段止盈止损配置表

| 阶段 | 普通止盈 | 移动止盈 | 能否平掉基础仓位 | 止损 | 网格分仓止盈 | 止盈止损模式 |
|:--|:--|:--|:--|:--|:--|:--|
| 空仓观察 | - | - | - | - | - | - |
| 等待首仓 | 关闭 | 关闭 | - | 关闭 | - | - |
| 首仓建立中 | 关闭 | 关闭 | **禁止** | 开启 | 关闭 | 仅止损 |
| 网格管理期 | 关闭 | 关闭 | **禁止** | 开启 | **条件开启** | 仅止损(未触发) / 网格分仓止盈(已触发) |
| 利润收割期 | 可开 | 可开 | **允许** | 开启 | 开启 | 完整止盈止损 |
| 风险退出期 | - | - | **允许** | 开启 | 关闭 | 止损优先 |

## 七、网格运行效率评估

| 评估维度 | 健康信号 | 警惕信号 |
|:--|:--|:--|
| 触发频率 | 每周 1-3 次触发 | 连续 2 周无触发（ATR 下降或支撑结构变化，重新计算间距） |
| 资金使用率 | 保持在 40%~70% | > 85%（接近打满，风险放大） |
| 综合成本线 | 趋势向有利方向移动 | 成本线持续向不利方向移动且无反弹迹象 |
| 盈亏平衡线距离 | 当前价格距盈亏平衡线 < 网格间距的 2 倍 | 价格远离盈亏平衡线且无回归迹象 |

## 八、决策稳定性原则

**核心命令：除非基本面反转或系统性风险确认，严禁因短期噪声频繁推翻执行中的逻辑。**

稳定性测试：当你准备改变策略阶段的判断时，先问自己——「我是否仅仅因为过去 24 小时的几个数据点而改变判断？如果过去 24 小时的价格数据不准确（网络故障），我的判断还成立吗？」

---

## 九、舆情校准原则

社交媒体情绪变化极快，单次波动不构成反转信号。

**仅当以下情况出现时才将舆情作为调整判断的依据：**

1. 出现连续 3 日以上的结构性情绪方向改变
2. 有实质性利空/利多事件支撑情绪变化（新闻确认，非传闻）
3. 社交媒体情绪 + 机构评级 + 价格走势三者同向共振

**舆情永远不能替代支撑压力位分析。** 舆情极端时可用作反向指标，但不能单独作为进出场依据。

```json
}
```

> **核心思路：** 马丁格尔网格在趋势中的先天缺陷是网格资金只赚小区间（每格 +3% 就走），在大波段中资金利用率极低。趋势锁仓模式通过在真实趋势确认时关闭网格减仓，让全部仓位参与趋势，把资金利用率从基础仓的 20-25% 提升到 70%+。退出判断同样基于 AI 对趋势减弱的综合感知——横盘、外部信息转向、逼近关键位、利润快速回吐等——而非单一的阈值条件。
