# 智能马丁格尔网格策略 — 退出规则

> 适用阶段：利润收割期、风险退出期，以及任何阶段需判断是否退出时
> 路由文档：[00-Router.md](00-Router.md)

---

## 一、禁止提前退出的核心原则

当策略有持仓但处于：首仓建立中、网格管理期（含未触发和已触发子状态）、已用资金占比显著低于建议仓位占比、网格尚未充分触发——默认目标只有四个字：

**「等，别动。」**

具体目标：保护基础仓、在网格管理期中运行、完成仓位建设。防止首仓买高、防止过早卖出导致网格无法启动。

---

## 二、允许进入利润收割期的条件

满足以下**任一**条件即可进入利润收割期：

| 序号 | 条件 | 来源 |
|:--:|:--|:--|
| 1 | **价格优先**：综合浮盈 ≥ 8% 且距一级压力位 < 2%（做多）/ 距一级支撑位 < 2%（做空） | 流转 5 |
| 2 | **网格充分**：网格触发 ≥ 2 次 + 综合浮盈 ≥ 10% | 流转 5a |
| 3 | **利润达标**：综合浮盈 ≥ 网格止盈 × 3 | 流转 5b |
| 4 | **AI 综合判断**：AI 基于外部信息（分析师目标价接近→product-all-info 评级、利好兑现→product-all-info 新闻、情绪极端化、板块轮动→大盘缓存等）判断趋势可能见顶/见底 | 置信度 ≥ 0.82 时可建议提前进入利润收割期 |

**利润收割期 = 只出不进，操作方向锁定为「空」（做多时出货、做空时平空仓）。** 若发现趋势还有延续空间、想继续加仓，应先流转回「网格管理期」，在那个阶段再操作。

### 2.1 综合浮盈计算公式

所有涉及 `综合浮盈` 的判定，统一按加权平均持仓成本计算，统合基础仓位与所有网格加仓的盈亏：

**做多：**

```
综合浮盈 = (当前价格 - 加权平均持仓成本) / 加权平均持仓成本
```

**做空：**

```
综合浮盈 = (加权平均持仓成本 - 当前价格) / 加权平均持仓成本
```

**加权平均持仓成本：**

```
加权平均持仓成本 = Σ(每笔仓位的股数 × 该笔成交价) / Σ(每笔仓位的股数)
```

**设计逻辑：**

- 做多时，加仓在价格下跌中发生，加权成本被逐步**摊低**（< 首仓价格）。加权平均后的综合浮盈大于仅计算基础仓的浮盈。
- 做空时，加空仓在价格上升中发生，加权成本被逐步**摊高**（> 首空仓价格）。加权平均后的综合浮盈同样大于仅计算基础空仓的浮盈。
- 综合浮盈反映的是**整个策略账户**的盈亏状态，包括基础底仓和所有网格仓位的平均效果，不是任何单一仓位的盈亏。

## 四、风险退出条件（完整 10 条）

### 事后确认型（原有 7 条）

1. 基本面发生明确重大的反向变化（引用 product-all-info 新闻/财报，不是传闻，是确认信息）
2. 趋势高置信度反转，置信度 > 0.75
3. 当前亏损超过止损幅度
4. 未来一周存在重大不可控事件（全局事件→calendar，个股事件→product-all-info 新闻），且风险收益比显著不利
5. **关键位失效**：做多时价格跌破一级支撑位且持续不收回到支撑位上方；做空时价格涨破一级压力位且持续不收回到压力位下方。短期回补/回落概率低
6. 网格已充分运行并进入利润收割期（正常流程结束，不是退出）
7. 已用资金占比接近最大仓位占比，继续持有风险收益比已恶化

### 先行预警型（AI 综合判断权重最高）

满足以下任一条件，或 AI 综合外部信息 + 技术面判断置信度 ≥ 0.82 认为风险显著上升，即可启动风险评估：

8. **多指标背离共振** — RSI、MACD、成交量中至少两个同时出现顶部/底部背离，且置信度 ≥ 0.70
9. **波动率异常放大** — ATR(14) 单日值放大至近 14 日均值的 1.8 倍以上，且价格方向不利
10. **关键位衰竭信号** — 价格触及一级压力位/支撑位后出现明显反转 K 线形态（长上影线/下影线 ≥ 实体 2 倍、吞没形态、黄昏之星/启明星），而后价格开始反向运行
11. **AI 外部信息预警** — 重大基本面恶化（product-all-info 新闻）、政策突变（大盘缓存）、流动性危机（大盘缓存）、竞争格局逆转（product-all-info 新闻）、信用评级下调（product-all-info 评级）等。AI 必须提供可验证的具体证据（新闻链接、财报数据、机构公告等），不得仅凭模糊情绪判断

**凡不满足以上任一条件，不得进入风险退出期。**

## 六、止损幅度计算

### 6.1 静态计算公式

止损幅度与网格参数直接绑定：

```
止损幅度 = 网格间距 × 网格数量 × 0.6
```

**设计逻辑：**

1. **用到 60% 网格即止损**——系数 0.6 表示，只需网格深度的 60% 即触发止损
2. **锚定持仓成本**——止损幅度基于当前持仓成本计算
   - 做多：加仓摊低了成本，止损触发时相对于首仓的实际跌幅已超过网格深度的 60%
   - 做空：加仓摊高了成本，止损触发时相对于首仓的实际涨幅已超过网格深度的 60%
3. **网格越深，止损越宽**——网格数量增多 = 容忍的波动幅度增大

**约束检查：** 做多时需验证 `持仓成本 × (1 - 止损幅度) < 一级支撑位`；做空时需验证 `持仓成本 × (1 + 止损幅度) > 一级压力位`。不满足时应增加网格数量或放宽间距。

### 6.2 动态收紧机制

当市场波动率出现异常放大时，静态止损可能反应不足。触发以下条件时，止损幅度临时收紧：

```
临时止损幅度 = 网格间距 × 网格数量 × 0.6 × 0.7
```

**触发条件：** ATR(14) 当前值 ≥ 近 14 日 ATR 均值的 1.5 倍。

**设计逻辑：** 波动率突然放大意味着价格可能快速穿越网格，风险加剧。临时收紧 30% 让止损更快响应异常市况。当 ATR(14) 回落至均值的 1.5 倍以下并保持 2 个交易日时，恢复原始止损幅度。

### 6.3 示例

**静态止损：**
```
间距 2%，8 档，首仓 $100
止损幅度 = 2% × 8 × 0.6 = 9.6%

用到第 5 档时：
  持仓成本 ≈ $94
  止损价 = $94 × (1 - 9.6%) = $84.98
  相对于首仓实际跌幅 = 15.02%
```

**动态收紧对比：**
```
ATR(14) 当前 = $4.50，近 14 日均值 = $2.80（放大 1.61×，触发收紧）

原始止损幅度 = 9.6%
临时止损幅度 = 9.6% × 0.7 = 6.72%

用到第 5 档时：
  原始止损价 = $84.98（距首仓 -15.02%）
  临时止损价 = $94 × (1 - 6.72%) = $87.68（距首仓 -12.32%，提前 2.7% 触发）
```
