# 大盘与板块和资金流向分析 模块

## 模块定位

本模块是 `tiger-stock-strategy-analysis` 技能的子模块，负责对**不同交易市场**进行大盘走势、板块轮动和资金流向的综合分析。

根据当前连接的市场类型，加载对应的分析文档作为 AI 执行指令。所有市场的缓存 key 统一由下方的「市场子模块」表格定义，**不得自定义 key 格式或路径**。

---

## 执行流程（AI 必须严格按此顺序执行）

### 第 1 步：查询缓存

从下方「市场子模块」表格的「缓存 key」列取出当前市场的完整 key，执行查询。例如美股：

```bash
python scripts/vnpy_command.py get - "/vnpy:美股:大盘与板块和资金流向分析"
```

- **替换规则**：将命令中的 `/vnpy:美股:大盘与板块和资金流向分析` 替换为表格中对应市场的完整 key
- key 以 `/` 开头表示公共数据，不归属任何用户

### 第 2 步：判断时效

- 如果缓存存在 **且** 距今 **≤ 12 小时** → **直接完整返回缓存 JSON 的原始数据，本模块执行结束。禁止重新分析。**
- 如果缓存不存在 **或** 距今 **> 12 小时** → 继续第 3 步

### 第 3 步：获取市场数据

使用 `get_market_data.py` 获取结构化价格数据：

```bash
# 美股示例
python scripts/get_market_data.py --market us_stocks --batch us-all --output json

# 加密货币示例
python scripts/get_market_data.py --market crypto --batch crypto-all --output json
```

> 可用 `--list-batches` 查看当前市场所有预设批次。
>
> **ICI 资金流数据**（美股专用）：
> ```bash
> # 股基+ETF 综合净流入 + 货币基金 AUM（一次获取）
> python scripts/get_market_data.py --fetch-url all --output json
> ```
>
> **全局经济日历**（所有市场共享，每小时 1 次）：
> ```bash
> # 非农/CPI/FOMC/央行利率等全部经济事件
> python scripts/get_market_data.py --fetch-url calendar --output json
> ```

### 第 4 步：执行分析

加载对应市场的分析文档（见下方「市场子模块」表格），按文档中的执行流程逐步分析。

> **⚠️ 输出格式铁律**：分析过程在内部完成，**不得向用户输出分析过程的中间文本**。最终只输出【六、输出模板】规定的 JSON 结果。如果分析文档的 Step 中有「结论输出」要求，那是要求填入 JSON 字段，不是输出给用户看。

### 第 5 步：保存结果

分析完成后，将 JSON 结果写入 Redis 缓存。从下方「市场子模块」表格的「缓存 key」列取出当前市场的完整 key，执行保存。例如美股：

```bash
python scripts/vnpy_command.py --token TOKEN publish 用户名 '/vnpy:美股:大盘与板块和资金流向分析' '{json结果}' --expire 43200
```

- **替换规则**：将命令中的 `/vnpy:美股:大盘与板块和资金流向分析` 替换为表格中对应市场的完整 key
- `--expire 43200` = 12 小时过期，与缓存时效一致
- **⚠️ key path 必须严格使用下方表格中定义的完整 key，不得自定义 key 格式或路径。存错位置会导致缓存检查失效，定时任务反复重跑。**

### 第 6 步：返回结果

将分析获得的 JSON 结果返回给用户。

> **⚠️ 输出铁律**：
> - JSON 格式**必须严格遵循**子文档【六、输出模板】中定义的完整结构
> - **所有字段必须保留**，即使数据不可得也要填入 `missing` 或 `null`，不得删除字段
> - 字段名**必须与模板完全一致**（中文 key），不得自行改用英文或其他命名
> - 输出前逐项核对子文档的输出前自检清单，确认无遗漏
