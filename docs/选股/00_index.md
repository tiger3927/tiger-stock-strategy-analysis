# 美股选股模块（AI 智能体作业指导书）

> **文档定位**：本文档是 `tiger-stock-strategy-analysis` 技能的子模块。当用户请求美股选股（做多或做空）机会分析时，本技能会加载此文档作为 AI 的执行指令。AI 应根据用户指定的 `direction` 参数选择做多或做空分支，严格按本文档的流程逐步执行，最终输出结构化 JSON 结果。
>
> **⚠️ 输出格式铁律**：最终输出必须是【六、输出模板】规定的 JSON 格式。分析过程中的中间结论是用于填入 JSON 字段的，**不得直接输出给用户**。用户只看到最终 JSON 结果。

---

## 一、模块目标

根据用户指定的方向（`direction`），加载对应的选股逻辑文档：

- **做多方向**：参照 [美股做多选择.md](美股做多选择.md)，结合当前大盘环境，**预测下一个轮入板块**，从候选股池中筛选出符合「价值+成长混合（GARP）」策略的做多标的，输出评分卡和入场计划。
- **做空方向**：参照 [美股做空选择.md](美股做空选择.md)，结合当前大盘环境，**预测下一个资金流出板块**，在估值泡沫、基本面恶化或板块轮动过热的标的中筛选做空标的，输出评分卡和入场计划。

回答五个核心问题：

```text
1. 当前大盘环境是否适合交易？  →  复用大盘分析缓存
2. 下一个可能轮动/流出的板块是什么？ →  板块轮动预测
3. 哪些标的值得关注？          →  候选池筛选
4. 每个标的的评分和入场计划？  →  评分卡打分
5. 风险在哪里？               →  风险提示
```

---

## 二、执行流程（AI 请严格按此顺序执行）

### Step 0：获取用户输入

在执行任何数据获取前，必须先与用户确认以下参数。如果用户已在上一次对话中提供，直接引用，无需重复询问。

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `direction` | **做多** / **做空** | 做多 |
| `time_frame` | 短期(1-4周) / 中期(1-6个月) / 长期(6个月+) | 短期 |
| `style` | 价值型 / 成长型 / 动量型 / 价值+成长混合 | 价值+成长混合 |
| `risk_tolerance` | 保守 / 中等 / 激进 | 中等 |
| `sector_preference` | 指定板块或"全市场" | 全市场 |
| `max_candidates` | 最终输出标的数上限 | 3 |

> **`direction` 路由规则**：
> - `direction = 做多` → 执行【做多方向】分支，参照 [美股做多选择.md](美股做多选择.md)
> - `direction = 做空` → 执行【做空方向】分支，参照 [美股做空选择.md](美股做空选择.md)
> - 如果用户未明确指定 `direction`，但表达中含有"做空/short/看跌"等语义 → 自动路由到做空分支

### Step 0.5：检查缓存

在执行任何数据获取之前，先检查 Redis 缓存中是否已有选股分析结果。

```bash
# 做多方向
python scripts/vnpy_command.py get 用户名 "/vnpy:美股:做多选股分析结果"

# 做空方向
python scripts/vnpy_command.py get 用户名 "/vnpy:美股:做空选股分析结果"
```

- 根据 `direction` 选择对应的缓存 key
- 如果缓存存在 **且** 距今 **≤ 24 小时** → **直接完整返回缓存 JSON，本模块执行结束。禁止重新分析。**
- 如果缓存不存在 **或** 距今 **> 24 小时** → 继续 Step 1

> 缓存 key 以 `/` 开头表示公共数据，不归属任何用户。`get` 命令中的 `-` 是用户名占位符（公用 key 无需具体用户名），读取时会被忽略。

### Step 1：获取大盘环境

**不重新分析大盘**，直接复用已有的大盘分析缓存。

```bash
python scripts/vnpy_command.py get 用户名 "/vnpy:美股:大盘与板块和资金流向分析"
```

- 如果缓存存在 → 解析 JSON，提取 `执行层摘要`（direction / risk_budget）、`板块轮动`（主线板块）、`风险提示` 等关键字段
- 如果缓存不存在 → 先执行大盘分析模块（按 [大盘与板块和资金流向分析/00_index.md](../大盘与板块和资金流向分析/00_index.md) 的流程），再取结果

**大盘环境判断规则**：

#### 【做多方向】

| 大盘方向 | 对选股的影响 |
|---------|-------------|
| 看多 + risk_budget ≥ 60 | 正常选股，可执行买入 |
| 震荡 + risk_budget 40-60 | 降低仓位上限，提高评分门槛（> 4.0 才执行） |
| 看空 + risk_budget < 40 | **不执行选股**，输出「大盘环境不适合做多」 |

#### 【做空方向】

| 大盘方向 | 对选股的影响 |
|---------|-------------|
| 看空 + risk_budget < 40 | 正常做空，可执行做空 |
| 震荡 + risk_budget 40-60 | 降低做空仓位上限，提高评分门槛（> 4.0 才执行做空） |
| 看多 + risk_budget ≥ 60 | **不执行做空**，输出「大盘环境不适合做空」 |

> **做空方向补充**：即使大盘看多，如果存在明确的板块过热信号（主线板块 20 日涨幅超额 > 10%、轮动阶段 = 晚期），仍可执行做空，但需降低仓位并提高评分门槛（> 4.5 才执行）。

### Step 1.5：预测下一个轮动板块

在确定候选池之前，先判断资金下一个可能流入的板块。**预测结果直接影响 Step 2 的候选池方向**。

#### 1.5.1 读取板块轮动状态

从大盘缓存中提取以下字段：

| 字段 | 用途 |
|------|------|
| `板块轮动.主线板块` | 当前资金主攻方向 |
| `板块轮动.流出板块` | 资金正在撤离的方向 |
| `板块轮动.轮动阶段` | 早期/中期/晚期 |
| `板块轮动.命中标准` | 判定阶段的依据 |
| `宏观数据.宏观结论` | 当前宏观环境定位 |
| `执行层摘要.risk_budget` | 风险预算，影响预测置信度 |

#### 1.5.2 获取板块相对强弱数据

```bash
python scripts/get_market_data.py --market us_stocks --batch us-sectors --output json
```

从结果中提取各板块 ETF（XLK/XLF/XLV/XLE 等）的以下数据：

| 数据 | 用途 |
|------|------|
| `change_pct`（当日涨跌幅） | 短期资金流向 |
| `pct_52w`（52周百分位） | 板块处于历史什么位置 |
| 近 20 日收益率（自行计算） | 中期相对强弱 |
| 成交量趋势 | 资金是否在加速流入 |

#### 1.5.3 轮动预测判断矩阵

AI 按以下矩阵判断下一个可能轮动/流出的板块。**满足 2 条及以上即可判定**。

##### 【做多方向】预测下一个轮入板块

| 当前状态 | 可能轮入的板块 | 判断依据 |
|---------|---------------|---------|
| 主线板块 20 日涨幅超额 > 8%（过热） | 同一大逻辑下的二线板块 | 资金溢出效应：AI 过热 → 半导体/软件补涨 |
| 轮动阶段 = 晚期 | 防御板块（医疗/必需消费/公用事业） | 资金从风险偏好转向避险 |
| 宏观结论 = Soft Landing | 金融/工业/材料 | 经济复苏预期利好周期 |
| 宏观结论 = 衰退风险 | 医疗/必需消费/公用事业 | 防御性配置 |
| 宏观结论 = 通胀反复 | 能源/材料/大宗商品 | 通胀受益板块 |
| 利率预期下行（10Y 回落） | 成长股/科技/房地产 | 利率敏感板块率先反弹 |
| 某板块 20 日跑输 SPY > 5% 但基本面未恶化 | 该板块本身 | 超跌反弹机会 |
| 某板块 ETF 成交量近 5 日放大 > 20% 但价格未涨 | 该板块 | 资金悄悄吸筹 |

##### 【做空方向】预测下一个资金流出板块

| 当前状态 | 可能流出的板块 | 判断依据 |
|---------|---------------|---------|
| 主线板块 20 日涨幅超额 > 10% | 该板块本身 | 短期过热，获利盘回吐 |
| 轮动阶段 = 晚期 | 当前主线板块 | 资金即将转向防御 |
| 宏观转向紧缩 | 高估值成长/科技 | 利率上行压制估值 |
| 经济衰退确认 | 周期板块（工业/材料/能源） | 需求下滑 |
| 板块 ETF 放量滞涨 | 该板块 | 资金出货 |
| 龙头股业绩不及预期 | 该板块及关联板块 | 龙头失速拖累整体 |
| 板块整体估值创历史新高（PEG > 2.0） | 该板块 | 估值无基本面支撑，回归压力大 |
| 利率预期上行（10Y 走高） | 成长股/科技/房地产 | 利率敏感板块率先承压 |

**板块轮动预测口诀**（仅作阶段感参考，不作单独判断依据）：

```text
【做多】AI领涨过热 → 半导体/软件补涨 → 金融接棒 → 工业/材料跟上 → 能源补涨 → 防御板块收尾
【做空】AI领涨过热 → 获利盘回吐 → 资金转向防御 → 高估值成长承压 → 周期股需求下滑
利率下行 → 成长/科技/地产率先反弹
利率上行 → 成长/科技/地产率先承压
经济衰退 → 医疗/消费/公用事业防御（做多）；周期股做空（做空）
```

> ⚠️ 该口诀仅为历史规律的简化描述，不可机械套用。实际判断必须结合矩阵中的可检验标准。

#### 1.5.4 预测输出

```json
{
    "预测板块": "金融",
    "置信度": "中",
    "预测依据": [
        "当前主线AI板块20日涨幅超额>8%，进入轮动晚期",
        "金融板块20日相对SPY超额-2%，处于滞后状态",
        "宏观环境高利率更久，利好金融板块净息差",
        "金融板块ETF(XLF)成交量近5日放大15%"
    ],
    "候选方向": "核心+卫星",
    "风险提示": "如果AI板块继续强势不回调，轮动可能推迟"
}
```

#### 1.5.5 预测结果对后续流程的影响

##### 【做多方向】

| 预测置信度 | 对 Step 2 候选池的影响 |
|-----------|----------------------|
| **高** | 候选池以预测板块为主（70%），当前主线板块为辅（30%） |
| **中** | 候选池同时覆盖预测板块和当前主线板块（各 50%） |
| **低** | 候选池以当前主线板块为主，预测板块仅作补充 |

##### 【做空方向】

| 预测置信度 | 对 Step 2 候选池的影响 |
|-----------|----------------------|
| **高** | 候选池以预测流出板块为主（70%），其他过热板块为辅（30%） |
| **中** | 候选池同时覆盖预测流出板块和其他过热板块（各 50%） |
| **低** | 以当前主线板块的弱势股为主，预测流出板块仅作补充 |

> **预测失败的处理**：如果预测的板块后续未按预期轮动（做多）或流出（做空），属于正常现象。板块轮动预测本身是概率判断，不是确定性预测。AI 应在输出中如实标注置信度，并在 `核心冲突` 中列出反向信号。

### Step 2：确定候选股池

从 [选股操作手册.md](选股操作手册.md) 中获取候选股池。

**候选池生成规则**（按优先级）：

1. **Step 1.5 预测板块** — 如果预测置信度为高或中，优先使用预测板块的候选股
2. **当前主线板块** — 如果无明确预测或预测置信度低，使用当前主线板块
3. **用户指定板块** — 如果用户指定了板块偏好，优先使用该板块的候选股
4. **全市场扫描** — 以上均不适用时，按 `选股操作手册.md` 方式二全网扫描

> **做空方向适配**：候选池来源以预测流出板块和其他过热板块为主。在 `选股操作手册.md` 的搜索关键词中，做空方向应使用高估/泡沫相关的搜索词（如 `"overvalued stocks high P/E bubble 2026"`），而非做多方向的低估/增长关键词。

**候选池大小控制**：总候选数控制在 20-40 只。如果超出，按市值从大到小截断。

### Step 3：获取候选池结构化数据

使用 `get_market_data.py` 获取候选池中所有标的的价格数据。

```bash
python scripts/get_market_data.py --market us_stocks --tickers TICKER1,TICKER2,... --output json
```

> 如果候选池超过 30 只，分两次获取。

**获取的数据字段**：price, change_pct, ma_20, ma_50, ma_200, high_52w, low_52w, pct_52w, volume

### Step 4：基本面快速过滤

对候选池中的每只标的，搜索基本面指标。**优先批量搜索**，减少调用次数。

```bash
# 批量搜索候选股的基本面数据（PE、PEG、营收增速、EPS增速等）
web_search "{ticker1}, {ticker2}, {ticker3} PE ratio PEG revenue growth EPS growth 2026"
```

**过滤规则**（满足以下至少 3 条即进入下一轮）：

##### 【做多方向】价值+成长混合（GARP）标准

| 指标 | 做多标准 |
|------|---------|
| P/E（静态/动态） | 低于行业中位数或自身历史均值 |
| PEG | < 1.5（理想 < 1.0） |
| 营收增速（YoY） | > 10%（连续 2-3 季度） |
| EPS 增速（YoY） | > 15% 或同比增速加快 |
| 自由现金流收益率 | > 4% |
| 净债务/EBITDA | < 2x（或净现金状态） |

##### 【做空方向】估值泡沫+基本面恶化标准

| 指标 | 做空标准 |
|------|---------|
| P/E（动态） | 高于行业中位数 2 倍以上 |
| P/S | > 10（非科技）或 > 20（科技） |
| PEG | > 2.0 且 EPS 增速放缓 |
| 营收增速（YoY） | 连续 2-3 季度放缓或转负 |
| EPS 增速（YoY） | 连续 2 季度不及预期或同比下滑 |
| 市值/FCF | > 50x（FCF 收益率 < 2%） |
| 净债务/EBITDA | > 5x（高杠杆） |

**提前退出规则**：如果基本面过滤后剩余标的 < 3 只，从候选池中补充市值最大的 10 只重新过滤。

### Step 5：技术面过滤

对通过基本面过滤的标的，按以下规则评分（每项 1-5 分）。

##### 【做多方向】

| 技术面指标 | 5分（强） | 3分（中性） | 1分（弱） |
|-----------|---------|-----------|---------|
| 趋势形态 | 价格在 MA20/MA50 上方，均线多头排列 | 价格在 MA20 附近，均线走平 | 价格在 MA50 下方，均线空头 |
| 成交量 | 上涨放量、回调缩量 | 成交量无明显异常 | 放量下跌或缩量上涨 |
| RSI(14) | 30-50 超卖反弹或 50-65 健康上升 | 50-70 中性偏强 | > 70 超买或 < 30 且继续下行 |
| 52周百分位 | 20-50 分位（低位启动） | 50-70 分位（中位） | > 80 分位（高位）或 < 10 分位（弱势） |
| 相对强度 | 近 20 日跑赢 SPY > 5% | 跑赢/跑输 SPY 在 ±5% 内 | 跑输 SPY > 5% |

**技术面总分** = 五项得分平均。总分 ≥ 3.0 进入下一轮。

##### 【做空方向】

| 技术面指标 | 5分（强做空信号） | 3分（中性） | 1分（弱/不适合做空） |
|-----------|----------------|-----------|-------------------|
| 趋势形态 | 价格在 MA50 下方，均线空头排列；50日线下穿200日线（死叉） | 价格在 MA20 附近，均线走平 | 价格在 MA20/MA50 上方，均线多头 |
| 成交量 | 上涨缩量、下跌放量；跌破关键位时放量确认 | 成交量无明显异常 | 上涨放量、回调缩量 |
| RSI(14) | > 70 超买区域；顶背离（价格新高、RSI未新高） | 50-70 中性偏强 | < 30 超卖或底背离 |
| 52周百分位 | > 80 分位（高位） | 50-70 分位（中位） | 20-50 分位（低位）或 < 10 分位 |
| 相对强度 | 近 20 日跑输 SPY > 5% | 跑赢/跑输 SPY 在 ±5% 内 | 近 20 日跑赢 SPY > 5% |

**技术面总分** = 五项得分平均。总分 ≥ 3.0 进入下一轮。

### Step 6：深度尽调与催化剂验证

对通过技术面的标的，逐一获取以下信息：

```bash
# 获取单品种新闻+评级（替代 web_search）
python scripts/get_market_data.py --fetch-url product-all-info --ticker TICKER --output json
```

同时用 `web_search` 补充以下信息：

| 信息维度 | 搜索指令示例 |
|---------|-------------|
| 最新财报 | `"{ticker} latest earnings results 2026"` |
| 分析师评级 | `"{ticker} analyst rating upgrade downgrade 2026"` |
| 内幕交易 | `"{ticker} insider buying selling 2026"` |
| 催化剂事件 | `"{ticker} catalyst product launch FDA approval 2026"` |
| 财报日期 | `"{ticker} earnings date 2026"` |

**深度尽调评分规则**（每项 1-5 分）：

##### 【做多方向】

| 维度 | 5分 | 3分 | 1分 |
|------|-----|-----|-----|
| 财报质量 | 超预期 + 上调指引 | 符合预期 | 不及预期 + 下调指引 |
| 催化剂强度 | 近 30 天内有明确正面催化剂 | 1-3 个月内有催化剂 | 无明显催化剂 |
| 分析师情绪 | 多家上调目标价/评级 | 维持评级 | 下调评级 |
| 内幕交易 | 管理层近期增持 | 无内幕交易 | 管理层大量减持 |

##### 【做空方向】

| 维度 | 5分（强做空信号） | 3分 | 1分（不利做空） |
|------|----------------|-----|---------------|
| 财报质量 | 不及预期 + 下调指引 | 符合预期 | 超预期 + 上调指引 |
| 催化剂强度 | 近 30 天内有明确负面催化剂（监管/诉讼/业绩miss） | 1-3 个月内有负面催化剂 | 无明显负面催化剂 |
| 分析师情绪 | 多家下调目标价/评级 | 维持评级 | 上调评级 |
| 内幕交易 | 管理层近期大量减持 | 无内幕交易 | 管理层近期增持 |
| 做空比例 | < 2%（低做空比例 + 高估值 = 无空头制约，轧空风险低） | 2-8% | > 15%（高轧空风险） |

### Step 7：评分卡综合打分

对通过深度尽调的标的，按以下权重计算综合评分。

##### 【做多方向】

| 维度 | 权重 | 评分来源 |
|------|------|---------|
| 价值低估 | 20% | Step 4 基本面过滤 |
| 增长潜力 | 25% | Step 4 基本面过滤 |
| 技术面形态 | 20% | Step 5 技术面过滤 |
| 催化剂强度 | 20% | Step 6 深度尽调 |
| 管理层/护城河 | 15% | Step 6 深度尽调 |

**综合判断规则**：

| 总分 | 操作建议 |
|------|---------|
| ≥ 4.0 | **可执行买入**，按入场计划执行 |
| 3.5 - 3.9 | **重点关注**，列入观察清单，等待催化剂触发 |
| < 3.5 | **暂不关注**，返回候选池 |

##### 【做空方向】

| 维度 | 权重 | 评分来源 |
|------|------|---------|
| 估值泡沫 | 20% | Step 4 基本面过滤 |
| 基本面恶化 | 25% | Step 4 基本面过滤 |
| 技术面形态 | 20% | Step 5 技术面过滤 |
| 下跌催化剂强度 | 20% | Step 6 深度尽调 |
| 管理层/资金面 | 15% | Step 6 深度尽调 |

**综合判断规则**：

| 总分 | 操作建议 |
|------|---------|
| ≥ 4.0 | **可执行做空**，按入场计划执行 |
| 3.5 - 3.9 | **重点关注**，列入观察清单，等待负面催化剂触发 |
| < 3.5 | **暂不关注**，返回候选池 |

**最终输出数量控制**：按总分从高到低排序，取前 `max_candidates` 只（默认 3 只）。

### Step 7.5：反方审查（Devil's Advocate Review）

> **定位**：在评分卡给出方向性结论后，反方审查负责**挑战这个结论**。如果反方证据足够强，下调评分或调整入场计划。这是防止"确认偏误"的关键步骤。
>
> - 【做多方向】在"看多"结论后，反方审查挑战看多逻辑
> - 【做空方向】在"看空"结论后，反方审查挑战看空逻辑，**特别警惕轧空风险**

#### 7.5.1 审查维度

对每只评分 ≥ 3.5 的标的，按以下维度逐一审查。

##### 【做多方向】8 个审查维度

| # | 审查维度 | 核心问题 | 数据来源 |
|---|---------|---------|---------|
| 1 | **涨幅已反映** | 近3个月涨幅是否已透支未来1-2个季度的利好？ | pct_52w、近期价格动量、RSI |
| 2 | **估值陷阱** | 低P/E是因为基本面在恶化吗？（区分"便宜"和"价值陷阱"） | EPS趋势方向、营收增速是否放缓 |
| 3 | **共识拥挤度** | 分析师推荐买入占比是否 > 80%？（过度拥挤是反向信号） | 分析师评级分布（consensus_rating） |
| 4 | **内幕交易反向** | 管理层近期是否在大幅减持？ | web_search insider selling |
| 5 | **技术面背离** | 价格创新高但成交量萎缩？RSI顶背离？ | 量价关系、RSI趋势 |
| 6 | **宏观逆风** | 是否存在该标的特有的宏观风险未被覆盖？ | 利率/汇率/政策影响 |
| 7 | **仓位集中度** | 最终输出中单一板块占比是否 > 40%？ | 评分卡结果汇总 |
| 8 | **催化剂已兑现** | 评分中依赖的催化剂是否已经发生（利好出尽）？ | 催化剂日期 vs 当前日期 |

##### 【做空方向】8 个审查维度

| # | 审查维度 | 核心问题 | 数据来源 |
|---|---------|---------|---------|
| 1 | **跌幅已反映** | 近3个月跌幅是否已透支未来1-2个季度的利空？（超跌反弹风险） | pct_52w、近期价格动量、RSI |
| 2 | **基本面反转** | 基本面是否出现改善信号？（营收/EPS增速触底回升） | EPS趋势方向、营收增速是否回升 |
| 3 | **轧空风险** | 做空比例是否 > 15%？（高做空比例 = 轧空炸弹） | FINRA Short Interest、IB 融券数据 |
| 4 | **内幕交易反向** | 管理层近期是否在增持？（内部人士看好 = 做空反向信号） | web_search insider buying |
| 5 | **技术面背离** | 价格创新低但成交量萎缩？RSI底背离？（下跌动能衰竭） | 量价关系、RSI趋势 |
| 6 | **宏观顺风** | 是否存在该标的特有的宏观利好未被覆盖？（降息/政策刺激） | 利率/汇率/政策影响 |
| 7 | **仓位集中度** | 最终输出中单一板块占比是否 > 40%？ | 评分卡结果汇总 |
| 8 | **催化剂已兑现** | 评分中依赖的负面催化剂是否已经发生（利空出尽）？ | 催化剂日期 vs 当前日期 |

#### 7.5.2 审查执行方式

```bash
# 对评分最高的 3-5 只标的，搜索内幕交易信息
web_search "{ticker} insider buying selling 2026"
```

#### 7.5.3 审查输出规则

| 反方证据强度 | 处理方式 |
|-------------|---------|
| **强**（满足 ≥ 2 条强证据） | 将该标的评分 **下调 0.5 分**，并在入场计划中标注"反方风险高" |
| **中**（满足 1 条强证据 或 ≥ 2 条弱证据） | 将该标的评分 **下调 0.3 分** |
| **弱**（满足 1 条弱证据） | 不下调评分，但在 `反方审查` 字段中记录质疑 |
| **无**（未发现反方证据） | 维持原评分 |

**强证据定义（做多方向）**：
- 涨幅已反映：近 3 月涨幅 > 30% 且 pct_52w > 90%（短期透支）
- 估值陷阱：EPS 连续 2 季度同比下滑但 P/E 偏低
- 共识拥挤度：> 80% 分析师评级为"买入"（过度乐观）
- 内幕交易反向：管理层近 30 日内净减持 > $1M
- 技术面背离：价格创新高但近 5 日均量 < 近 20 日均量的 80%
- 催化剂已兑现：评分依赖的核心催化剂日期已过且股价未涨

**强证据定义（做空方向）**：
- 跌幅已反映：近 3 月跌幅 > 30% 且 pct_52w < 10%（超跌反弹风险高）
- 基本面反转：EPS 连续 2 季度同比增速回升
- 轧空风险：做空比例 > 15% 且股价近 5 日放量上涨
- 内幕交易反向：管理层近 30 日内净增持 > $1M
- 技术面背离：价格创新低但近 5 日均量 < 近 20 日均量的 80%（下跌动能衰竭）
- 催化剂已兑现：评分依赖的核心负面催化剂日期已过且股价未跌

**弱证据定义（做多方向）**：
- 涨幅已反映：近 3 月涨幅 15-30% 或 pct_52w 80-90%
- 共识拥挤度：60-80% 分析师评级为"买入"
- 技术面背离：RSI > 70（超买但无背离）
- 仓位集中度：单一板块占比 > 40%

**弱证据定义（做空方向）**：
- 跌幅已反映：近 3 月跌幅 15-30% 或 pct_52w 10-20%
- 轧空风险：做空比例 8-15%
- 技术面背离：RSI < 30（超卖但无背离）
- 仓位集中度：单一板块占比 > 40%

#### 7.5.4 审查结果对后续流程的影响

##### 【做多方向】

| 审查结果 | 对 Step 8 入场计划的影响 |
|---------|----------------------|
| 评分未下调 | 正常执行 Step 8 |
| 评分下调 0.3 | 建仓方式改为"1 批轻仓"，仓位上限减半 |
| 评分下调 0.5 | 移出"可执行买入"列表，降级至"观察清单" |

##### 【做空方向】

| 审查结果 | 对 Step 8 入场计划的影响 |
|---------|----------------------|
| 评分未下调 | 正常执行 Step 8 |
| 评分下调 0.3 | 做空仓位减半，优先使用 put 期权而非直接融券 |
| 评分下调 0.5 | 移出"可执行做空"列表，降级至"观察清单"（警惕轧空） |

> **反方审查不是否定评分卡**，而是提供"另一面视角"。最终决策是正向评分 + 反方审查的综合结果。反方审查结果写入输出模板的 `反方审查` 字段。

### Step 8：生成入场计划

对评分 ≥ 3.5 的标的，生成入场计划。

##### 【做多方向】

| 字段 | 生成规则 |
|------|---------|
| `建仓区间` | 当前价 ± 3-5%，结合 MA20/MA50 支撑位 |
| `第一目标` | 前高或阻力位 |
| `第二目标` | 分析师共识目标价或 DCF 估值 |
| `止损位` | 入场价的 5-8% 下方，或跌破 MA50 |
| `仓位上限_pct` | 大盘 risk_budget 正常时 10-15%，偏低时 5-8% |
| `建仓方式` | 评分 ≥ 4.0 分 2 批建仓（1/3 底仓 + 回调加仓）；3.5-3.9 分 1 批轻仓 |

##### 【做空方向】

| 字段 | 生成规则 |
|------|---------|
| `建仓区间` | 反弹至 MA20/MA50 阻力位附近，而非追跌 |
| `第一目标` | 前低或支撑位 |
| `第二目标` | 基于 DCF 估值或可比公司 P/E 估值的合理价 |
| `止损位` | 入场价的 5-10% 上方（做空止损应更宽，防轧空），或突破 MA50 |
| `仓位上限_pct` | 大盘 risk_budget 正常时 5-8%，偏低时 3-5%（做空仓位小于做多） |
| `建仓方式` | 评分 ≥ 4.0 分 2 批建仓（1/3 底仓 + 反弹加仓）；3.5-3.9 分 1 批轻仓 |
| `做空工具` | **优先使用 put 期权**（亏损有限），其次融券做空 |
| `做空比例检查` | 做空前确认做空比例 < 15%，否则标注"高轧空风险"并降级处理 |

### Step 9：汇总输出

按【六、输出模板】生成结构化 JSON 结果。

> **⚠️ 禁止保存 JSON 文件到磁盘**：生成的 JSON 结果**只返回给用户 + 发布到 Redis**，不得在本地保留任何中间结果文件。如果 AI 有自动保存中间结果的习惯，请在此步骤关闭。

### Step 10：保存结果到缓存

将生成的 JSON 结果写入 Redis 缓存。由于选股结果内容较大（可能几千字），**必须使用 `--file` 参数**从临时文件读取，避免命令行长度限制。

临时文件统一存放在 `tests\tmp\` 目录下，发布后**必须删除**，避免残留过多：

```powershell
# 1. 将 JSON 结果写入临时文件
@'
{json结果}
'@ | Set-Content tests\tmp\stock_pick_result.json -Encoding utf8

# 2. 发布到 Redis（做多方向）
python scripts/vnpy_command.py --token TOKEN publish 用户名 /vnpy:美股:做多选股分析结果 --file tests\tmp\stock_pick_result.json --expire 86400

# 3. 清理临时文件
Remove-Item tests\tmp\stock_pick_result.json
```

做空方向同理，替换 key 为 `/vnpy:美股:做空选股分析结果`。

- `--expire 86400` = 24 小时过期，与缓存时效一致
- key 以 `/` 开头表示公共数据，不归属任何用户
- 根据 `direction` 选择对应的缓存 key
- 发布完成后**必须删除临时文件**，遵守"不保留中间结果"原则
- **注意**：如果中途中断（如 Ctrl+C），请手动执行 `Remove-Item tests\tmp\stock_pick_result.json` 清理残留文件

## 四、数据缺失处理规则

| 场景 | 处理方式 |
|------|---------|
| 某只标的的基本面数据无法获取 | 标记该标的 `fundamental_data_missing`，跳过该标的 |

### 输出前自检清单

AI 在输出前必须逐条检查以下 6 项：

| # | 检查项 | 要求 |
|---|--------|------|
| 1 | 字段完整性 | 输出 JSON 包含【六、输出模板】中的所有字段，无遗漏。数据不可得时填入 `missing` 或 `null`，不得删除字段 |
| 2 | 字段名一致性 | 所有字段名与模板完全一致，不得自行改用英文或其他命名 |
| 3 | ticker + name 成对出现 | 每个 `ticker` 字段后紧跟 `name` 字段 |
| 4 | 评分卡结果按 rank 排序 | rank 1 为最高分，依次排列 |
| 5 | 反方审查已执行 | 每只评分 ≥ 3.5 的标的都有 `反方审查` 字段，包含评分调整、审查结论和质疑项 |
| 6 | 核心冲突已列出 | 包含至少利多 + 利空两个方向，且最后一条为判断 |
| 7 | 无交易动作语言 | 全文未出现：建议买入/卖出、调仓、降仓、加仓、减仓、清仓 等交易指令词（`执行建议` 字段除外，该字段是给 AI 的决策参考） |
