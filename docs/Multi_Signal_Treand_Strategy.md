# 多信号权重评分趋势策略 (Multi_Signal_Treand_Strategy)

## AI角色定位

本策略的量化系统会自动执行交易逻辑，AI在此框架中承担以下三项职责：

### 1. 订单确认（Order Confirmation）
量化系统在**即将执行开仓前**，可能请求AI进行确认。AI收到请求后，需结合本文档的策略逻辑、当前状态信息和市场综合情况，判断该交易是否合理，并按请求指定的格式返回确认或否决意见。

### 2. 风险检查（Risk Check）
量化系统会**周期性**（如每根K线）向AI提交当前持仓和策略状态，请求风险分析。AI发现持仓存在重大风险（如异常波动、趋势逆转、浮亏过大等），可按请求指定的格式发送操作建议（如减仓、平仓、调整止盈止损等）。

### 3. 其他建议（Advisory）
AI不干涉量化系统的自主运行，但可在任何分析请求中提供建议，包括但不限于：
- 参数调整建议
- 品种切换建议
- 市场异常预警

> **核心原则**：量化系统自主执行交易，AI在系统请求的范围内提供分析和建议，不直接操作账户。

## 策略概述

> 📖 策略逻辑由代码自动执行，AI阅读此节以理解策略的运行机制，从而在【订单确认】和【风险检查】中做出准确判断。

基于 **EMA均线交叉 + 多指标置信度评分** 的趋势跟踪策略。采用 **RSI、ADX、CCI、ATR、EMA、DMI、布林带、K线形态** 8个维度的加权评分体系，对多空方向分别打分，选择置信度最高的方向开仓。

**适用品种**: 股票、期货、数字货币等趋势性品种
**操作方式**: 多空双边交易，每次满仓进出
**持仓周期**: 几天到一个月的中短期趋势

## 代码架构层级

> 📖 了解代码的层级关系，有助于AI理解数据来源和功能归属，便于在【风险检查】中定位问题。

TigerGridTemplate (基类)
  └─ 仓位管理、止盈止损、移动止盈、状态发布
      └─ Multi_Signal_Treand_Strategy (本策略)
          └─ 多指标评分、机会点检测、信号冷却

### 基类功能 (TigerGridTemplate)

基类功能较多，本策略只用如下功能：

- **仓位管理**: 通过 `set_target_pos()` / `get_all_canbuy_volume()` 管理持仓目标
- **固定止盈**: 持仓盈利达到设定的"止盈阈值"时平仓
- **固定止损**: 持仓亏损达到设定的"止损阈值"时平仓
- **移动止盈**: 盈利超过设定的"移动止盈启动幅度"后启动，跟踪持仓最高价，回撤达到"移动止盈回撤幅度"时平仓

## 核心执行流程

> 📖 策略自动执行流程，AI了解每个环节后，可在【订单确认】时评估开仓合理性，在【风险检查】时定位异常环节。

### 每根K线执行顺序

```
do_bar() 每根K线回调
  │
  ├─ 1. 冷却计数器递减（同方向信号冷却）
  │
  ├─ 2. 计算技术指标（追加到历史数组）
  │    ├─ 快EMA / 慢EMA
  │    ├─ ADX / +DI / -DI (DMI)
  │    ├─ RSI
  │    ├─ ATR (波动率)
  │    └─ CCI
  │
  ├─ 3. 判断EMA方向（锁存机制：方向只在变化时更新，持续维持）
  │    ├─ 快线 > 慢线 且间距≥CFS → 多头排列 (方向=1)
  │    └─ 快线 < 慢线 且间距≥CFS → 空头排列 (方向=-1)
  │
  ├─ 4. 计算多空置信度评分 (8项加权打分)
  │    ├─ 计算做多评分  → 得到做多分数、做多等级、做多趋势
  │    └─ 计算做空评分  → 得到做空分数、做空等级、做空趋势
  │
  ├─ 5. 检测开仓机会点
  │    ├─ 检测做多机会  → pullback(回踩反弹) / none
  │    └─ 检测做空机会  → pullup(回抽下跌) / none
  │
  ├─ 6. 选择最佳方向（比较多空信号）
  │
  ├─ 7. 判断是否满足开仓条件
  │    ├─ 最佳方向 ≠ none
  │    ├─ 置信度等级为 A+
  │    ├─ 机会点类型匹配 (做多=pullback，做空=pullup)
  │    ├─ 满足信号冷却（不同方向直接放行/同方向需等待冷却归零）
  │    └─ 非同方向持仓（已有持仓方向 ≠ 信号方向）
  │
  └─ 8. 执行开仓
       ├─ 打印信号日志（含置信度分数、等级、8项分项得分）
       ├─ 计算目标仓位（满仓）
       └─ 执行开仓下单
```

## 置信度评分体系（总分100分）

> 📖 评分由代码自动计算，AI了解评分规则后，可在【订单确认】时判断评分是否合理，在【风险检查】时发现评分异常。

### 1. RSI 得分 [满分30]

**做多（底部反弹）**:
| 条件 | 得分 |
|---|---|
| RSI连续2根线上升 | +15 |
| 前2根RSI最低值 < 40（超卖区域） | +10 |
| 当前RSI在40~65之间 | +5 |

**做空（高位反转）**:
| 条件 | 得分 |
|---|---|
| RSI连续2根线下降 | +15 |
| 前2根RSI最高值 > 60（超买区域） | +10 |
| 当前RSI在35~60之间 | +5 |

### 2. CCI 得分 [满分20]

**做多（底部反弹）**:
| 条件 | 得分 |
|---|---|
| CCI连续2根线上升 | +10 |
| 前2根CCI最低 < -80 且当前回升 | +10 |

**做空（高位反转）**:
| 条件 | 得分 |
|---|---|
| CCI连续2根线下降 | +10 |
| 前2根CCI最高 > 100 且当前回落 | +10 |

### 3. ATR 得分 [满分10]
| 条件 | 得分 |
|---|---|
| ATR比率在0.8~1.3（波动率稳定） | +5 |
| ATR比率 ≤ 0.85（波动率缩小，趋势前兆） | +5 |

### 4. ADX 得分 [满分10]
| 条件 | 得分 |
|---|---|
| ADX > 25（强趋势） | +5 |
| ADX在20~25（趋势酝酿中） | +3 |

### 5. EMA 得分 [满分10]

**做多**:
| 条件 | 得分 |
|---|---|
| 价格在快EMA附近1%以内（回踩确认） | +3 |
| 价格 > 快EMA（在均线上方） | +2 |
| EMA方向向上 | +5 |

**做空**:
| 条件 | 得分 |
|---|---|
| 价格在快EMA附近1%以内（回抽确认） | +3 |
| 价格 < 快EMA（在均线下方） | +2 |
| EMA方向向下 | +5 |

### 6. K线 得分 [满分5]

**做多**:
| 条件 | 得分 |
|---|---|
| 阳线（收盘 > 开盘） | +3 |
| 下影线 > 实体1倍（探底回升） | +2 |

**做空**:
| 条件 | 得分 |
|---|---|
| 阴线（收盘 < 开盘） | +3 |
| 上影线 > 实体1倍（冲高回落） | +2 |

### 7. 方向 得分 [满分5] — DMI确认
| 条件 | 得分 |
|---|---|
| 做多: +DI > -DI（多头主导，方向确认） | +5 |
| 做空: -DI > +DI（空头主导，方向确认） | +5 |

### 8. 布林 得分 [满分10]

**做多**:
| 条件 | 得分 |
|---|---|
| 价格 ≤ 中轨（偏低位置） | +5 |
| 价格距中轨 < 带宽30%（居中位置） | +3 |
| 价格距下轨 < 带宽20%（超卖位置） | +2 |

**做空**:
| 条件 | 得分 |
|---|---|
| 价格 ≥ 中轨（偏高位置） | +5 |
| 价格距中轨 < 带宽30%（居中位置） | +3 |
| 价格距上轨 < 带宽20%（超买位置） | +2 |

## 机会点检测规则

> 📖 机会点由代码自动检测，AI了解规则后可在【订单确认】时判断当前信号是否匹配机会点类型。

### 做多机会点 (pullback 回踩反弹)
需同时满足以下5个条件：
1. **EMA多头排列**: 快EMA在慢EMA上方
2. **回踩EMA**: 价格在快EMA附近 ±1.5%以内
3. **阳线确认**: 收盘价 > 开盘价
4. **动量反弹**: RSI连续2根线上涨且涨幅≥5点，或CCI连续2根线上涨
5. **非超买**: RSI ≤ 75 且 CCI ≤ 200

### 做空机会点 (pullup 回抽下跌)
需同时满足以下5个条件：
1. **EMA空头排列**: 快EMA在慢EMA下方
2. **回抽EMA**: 价格在快EMA附近 ±1.5%以内
3. **阴线确认**: 收盘价 < 开盘价
4. **动量回落**: RSI连续2根线下跌且跌幅≥5点，或CCI连续2根线下跌
5. **非超卖**: RSI ≥ 25 且 CCI ≥ -200

## 最佳方向选择逻辑

> 📖 方向由代码自动选择，AI了解逻辑后可在【订单确认】时验证方向选择是否合理。


```
如果 做多有机会 AND 做空有机会:
    选择 做多分数 和 做空分数 中较高的方向（相等时不操作）
如果 仅做多有机会:
    选择 做多
如果 仅做空有机会:
    选择 做空
否则:
    不操作，最佳方向 = none
```

## 信号冷却机制

> 📖 冷却由代码自动维护，AI了解后可避免对冷却期内的信号误判，在【风险检查】时了解信号频率是否异常。

```
检查冷却(direction):
  ├─ 方向与上次信号方向不同 → 直接放行 (多→空 或 空→多)
  └─ 方向与上次信号方向相同 → 检查冷却计数器
       ├─ 冷却计数器 == 0 → 放行
       └─ 冷却计数器 > 0 → 等待

每次开仓后: 冷却计数器 = 冷却K线数
每根K线:    冷却计数器 -= 1
```

## 开仓条件完整判断

> 📖 开仓条件检查和执行由代码自动完成。AI可在【订单确认】时综合以上规则判断开仓是否合理，在【风险检查】时评估持仓是否仍符合开仓逻辑。

开仓前需通过所有检查：
1. 最佳方向 ≠ "none"
2. 置信度等级为 **A+**（分数达到A+阈值）
3. 机会点类型匹配（做多必须为 pullback，做空必须为 pullup）
4. 冷却检查通过
5. 非同方向持仓：现有持仓方向与信号方向不同（避免同方向频繁微调）

通过后执行：
- 计算目标仓位 = 可用资金 × 开仓动用资金比例（满仓）
- 执行开仓下单，记录本次信号方向，重置冷却计数器
